## 五、更新流程

### 5.1 标准更新流程

```
1. 确定更新需求
   ├── 检查更新清单
   ├── 确定更新内容
   └── 确定优先级

2. 执行更新
   ├── 读取现有文件
   ├── 编写更新内容
   ├── 写入文件
   └── 更新SKILL.md和README.md

3. 验证更新
   ├── 检查文件完整性
   ├── 检查格式规范
   ├── 检查触发词
   └── 检查交叉引用

4. 记录更新
   ├── 更新版本历史
   ├── 更新记忆
   └── 通知用户
```

### 5.2 紧急更新流程

```
1. 发现问题
   ├── 用户反馈
   ├── 自我检查
   └── 外部变化

2. 评估影响
   ├── 影响范围
   ├── 紧急程度
   └── 修复方案

3. 快速修复
   ├── 定位文件
   ├── 修复内容
   └── 验证修复

4. 记录修复
   ├── 更新版本历史
   ├── 记录问题
   └── 防止复发
```
