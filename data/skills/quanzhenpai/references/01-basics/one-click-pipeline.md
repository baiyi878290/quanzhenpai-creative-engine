## 二、详细流程

### Phase 1: 信息提取（1轮对话）

**输入**：剧本/小说/故事大纲

**输出**：
```
1. 角色卡（所有角色）
   - 姓名、年龄、身份
   - 外貌特征（发型/五官/肤色/身材）
   - 标志性服装（不同场景）
   - 性格→视觉转化
   - 视觉锚点

2. 场景卡（所有场景）
   - 场景名称
   - 场景类型（室内/室外）
   - 色调（冷调/暖调/暗调）
   - 核心元素
   - 光影氛围

3. 道具卡（关键道具）
   - 道具名称
   - 材质
   - 颜色
   - 细节特征

4. 分镜脚本（全集）
   - 镜号、景别、运镜
   - 画面描述
   - 台词/音效
   - 时长
```

### Phase 2: 设定板生成（批量输出）

**输出**：

```
1. 角色设定板提示词（每个角色1个）
   格式：左右分割式（肖像+三视图+细节）

2. 场景设定板提示词（每个场景1个）
   格式：12要素完整描述

3. 道具设定板提示词（每个道具1个）
   格式：左右分割式（主视觉+三视图+细节）

4. 分镜故事板提示词（每4-6镜头1个）
   格式：2×2手绘线稿
```

**用户操作**：
```
1. 复制角色设定板提示词 → GPT-Image-2生成 → 保存为角色参考图
2. 复制场景设定板提示词 → GPT-Image-2生成 → 保存为场景参考图
3. 复制道具设定板提示词 → GPT-Image-2生成 → 保存为道具参考图
4. 复制分镜故事板提示词 → GPT-Image-2生成 → 保存为分镜参考图
```

### Phase 3: 提示词生成（批量输出）

**输出**：
```
Seedance提示词文件（每镜头1条）

格式：
## S001
- 角色参考：char_舒兰舟.png
- 场景参考：scene_实验室.png
- 分镜参考：storyboard_S001.png
- 提示词：
  Wide overhead shot, fixed camera,
  two scientists standing face to face in laboratory,
  seeming to argue about something,
  tense atmosphere,
  cinematic, 9:16
  Duration: 3 seconds
  台词: ""

## S002
...
```

### Phase 4: 视频生成（用户操作）

**操作步骤**：
```
对于每个镜头：
1. 打开Seedance 2.0
2. 上传角色参考图
3. 上传场景参考图
4. 上传分镜参考图（如有）
5. 粘贴提示词
6. 设置时长
7. 点击生成
8. 下载视频片段
```

### Phase 5: 后期合成（用户操作）

**工具**：剪映/Premiere/达芬奇

**操作**：
```
1. 导入所有视频片段
2. 按分镜顺序排列
3. 添加转场效果
4. 添加字幕
5. 添加音效/配乐
6. 导出最终视频
```

## 四、算力消耗估算

### GPT-Image-2 消耗

| 任务 | 数量 | 估算 |
|------|------|------|
| 角色设定板 | N个角色 | N张图 |
| 场景设定板 | M个场景 | M张图 |
| 道具设定板 | K个道具 | K张图 |
| 分镜故事板 | 每4-6镜头1个 | 总镜头数/5张图 |
| **总计** | | N+M+K+镜头数/5 |

### Seedance 2.0 消耗

| 任务 | 数量 | 估算 |
|------|------|------|
| 视频生成 | 每镜头1次 | 总镜头数次 |
| **总计** | | 总镜头数 |

### 示例：10集竖屏短剧

```
假设：
- 3个主要角色
- 5个场景
- 2个关键道具
- 每集4个镜头，共40个镜头

GPT-Image-2：
- 角色设定板：3张
- 场景设定板：5张
- 道具设定板：2张
- 分镜故事板：40/5=8张
- 总计：18张图

Seedance 2.0：
- 视频生成：40次
- 总计：40个视频片段
```

## 五、优化策略

### 5.1 减少GPT-Image-2消耗

```
1. 角色设定板：只生成主要角色（3-5个），次要角色用文字描述
2. 场景设定板：只生成核心场景（5-8个），重复场景复用
3. 道具设定板：只生成关键道具（2-3个），普通道具不生成
4. 分镜故事板：只生成关键镜头，简单镜头跳过
```

### 5.2 减少Seedance消耗

```
1. 先生成1个代表性镜头验证效果
2. 验证通过后再批量生成
3. 使用fast版本加速（牺牲少量质量）
4. 相同角色/场景的镜头复用参考图
```

### 5.3 减少对话轮次

```
1. Phase 1一次性输出所有信息卡
2. Phase 2一次性输出所有提示词
3. Phase 3一次性输出所有Seedance提示词
4. 总共3轮对话完成全部输出
```

## 六、质量控制

### 6.1 角色一致性检查

```
□ 所有镜头中角色外观一致
□ 服装在不同场景中正确切换
□ 发型、配饰保持一致
```

### 6.2 场景一致性检查

```
□ 同一场景的镜头光影方向一致
□ 色调保持统一
□ 核心元素位置稳定
```

### 6.3 风格一致性检查

```
□ 全片风格关键词统一
□ 画质参数一致
□ 时长分配合理
```

### 6.4 动作自然度检查

```
□ 动作描述简洁明确
□ 无复杂多动作叠加
□ 表情变化自然
```

## 七、触发词

| 触发词 | 功能 |
|--------|------|
| 一键成片 | 启动完整工作流 |
| 批量生成 | 批量输出所有提示词 |
| 生成设定板 | 生成所有设定板提示词 |
| 生成分镜 | 生成分镜脚本和故事板 |
| 生成Seedance提示词 | 生成所有镜头提示词 |

## 八、用户交互流程

```
用户：一键成片，[贴剧本内容]
AI：[一次性输出Phase 1+2+3所有内容]
用户：确认/修改
AI：[输出最终可直接使用的文件]
用户：复制到各平台生成
```

总共2轮对话，最小化算力消耗。
