# 全帧派 7步SOP 详解

> 设计者：季睿 | 2026-05-11

## 流程总览

```
帧妞选题 → 全崽快筛 → 小阿派DAG → 小阿派执行∥全崽风控 → 帧妞审核 → 全崽合成 → 全崽归档
```

## STEP 1: 帧妞选题

**目标**: 判断创作方向、赛道、视觉调性

**输入**: 用户原始需求
**输出**: 选题判断 + 调性锚定（1-3句话）

**执行**:
1. 加载帧妞面板 (PERSONA-帧妞.md)
2. 从视觉/节券/审美角度评估选题价值
3. 输出: 方向判断 + 核心调性关键词

**跳过条件**: 数据型/格式型/查询型任务

## STEP 2: 全崽可行性快筛（5分钟）

**目标**: 快速识别不可行因素，避免执行到一半发现走不通

**输入**: 帧妞的选题判断
**输出**: 绿/红灯 + 风险清单

**检查项**:
- 合规风险：政策红线、平台审核标准
- 版权雷区：IP冲突、相似作品
- 市场窗口：同期竞品、档期冲突
- 技术边界：当前工具/API能力是否支撑

**红灯触发**: 任一项≥高风险 → 回STEP 1

## STEP 3: 小阿派 DAG拆解

**目标**: 复杂任务分模块，定并行/串行

**输入**: 通过快筛的选题
**输出**: DAG任务图

**执行**:
1. 加载小阿派面板 + xiao-apai-master skill
2. 识别独立可并行模块
3. 识别串行依赖链
4. 估算各模块耗时

**跳过条件**: 单一任务（无并行机会）

## STEP 4: 小阿派执行 + 全崽并行风控

**目标**: 内容生成 + 不阻塞的异步质量检查

**小阿派执行**:
- 加载对应子技能（小说→novel-writing-pipeline, 分镜→dual-track-film-master, 改编→novel-film-adaptation）
- 按DAG顺序生成内容

**全崽并行风控**（异步、不阻塞主流程）:
- 一致性检查：人名/地名/时间线/设定自洽
- 合规扫描：敏感词/红线内容
- 质量flag：逻辑漏洞、前后矛盾、format偏差
- 输出：flag列表（标记位置+问题+严重等级）

## STEP 5: 帧妞质量审核

**目标**: 快速审核关键内容，决定通过/退回

**输入**: 小阿派执行结果 + 全崽风控flags
**输出**: ✅通过 / ❌退回(附具体问题)

**审核方式**:
- 只看flags标记的段落 + 关键节点（开头/转折/结尾）
- 不看全部（节省时间）
- 帧妞的判断依据：视觉节奏感 + 观众第一反应直觉

**退回上限**: 2轮
**超限处理**: 全崽硬裁 — 对比两版差异，帧妞一句话定案

## STEP 6: 全崽合成交付

**目标**: 最终合规检查 + 格式转换 + 打包发送

**执行**:
1. 最终合规扫描（全量）
2. 格式转换 → .docx（默认）
3. 排版美化（字体/字号/间距/页码）
4. 打包（多文件→.zip）
5. 通过message工具发送给用户

**多文件策略**: 有依赖关系的文件打包.zip，单文件直接发送

## STEP 7: 全崽归档

**目标**: 信息持久化，支持后续检索和优化

**执行**:
1. 更新 `memory/YYYY-MM-DD.md`（任务名称+状态+耗时+关键决策）
2. 更新 `MEMORY.md`（如产生可复用知识）
3. 更新 `references/lessons.md`（如有失误）
4. 检查是否需要更新技能文件

## 异常处理

| 场景 | 处理 |
|------|------|
| 执行中断（session重启） | 读memory/当日.md → 恢复STEP |
| 子技能缺失 | 触发skill-creator自动创建 |
| API/工具不可用 | 降级方案（本地替代/web_fetch→存量知识） |
| 用户手动召唤角色 | 锁定单角色，全帧派暂停自动路由 |
