# 平台算法深度指南

## 一、抖音算法详解

### 1.1 核心机制

```
流量池机制：

初级池（200-500播放）
    ↓ 完播率>30%，点赞>5%
二级池（1000-5000播放）
    ↓ 完播率>25%，点赞>4%
三级池（1万-10万播放）
    ↓ 完播率>20%，点赞>3%
四级池（10万-100万播放）
    ↓ 完播率>15%，点赞>2%
五级池（100万+播放）
```

### 1.2 核心指标权重

| 指标 | 权重 | 计算方式 | 优化方向 |
|------|------|----------|----------|
| 完播率 | 35% | 看完人数/播放人数 | 时长、节奏、钩子 |
| 点赞率 | 25% | 点赞数/播放数 | 内容质量、情感共鸣 |
| 评论率 | 20% | 评论数/播放数 | 话题性、争议性 |
| 转发率 | 10% | 转发数/播放数 | 实用性、传播性 |
| 关注率 | 10% | 关注数/播放数 | 人设、系列性 |

### 1.3 算法触发机制

```
发布后0-1小时：
├── 初级审核（机器）
│   ├── 内容合规检测
│   ├── 画面质量检测
│   └── 原创度检测
│
├── 初级推荐（200-500播放）
│   ├── 推荐给兴趣匹配用户
│   ├── 推荐给同城用户
│   └── 推荐给关注者
│
└── 数据观察（1-24小时）
    ├── 完播率
    ├── 互动率
    └── 转化率

发布后1-3天：
├── 进入二级池（如果数据好）
├── 进入三级池（如果数据很好）
└── 进入四级池（如果数据爆了）

发布后3-7天：
├── 流量逐渐衰减
├── 长尾流量
└── 进入合集/搜索流量
```

### 1.4 竖屏短剧特殊算法

```
短剧专属流量池：
├── 免费短剧流量池
│   ├── 完播率权重更高
│   ├── 单集时长1-3分钟
│   └── 系列剧加权
│
├── 付费短剧流量池
│   ├── 付费率权重最高
│   ├── 前3集免费
│   └── 付费点设计关键
│
└── 短剧推荐机制
    ├── 完结剧优先
    ├── 高评分剧优先
    └── 更新频率稳定优先
```

### 1.5 发布时间优化

| 时段 | 用户活跃度 | 适合内容 |
|------|------------|----------|
| 7:00-9:00 | 高 | 通勤、早间 |
| 12:00-13:00 | 极高 | 午休、碎片时间 |
| 18:00-20:00 | 极高 | 下班、晚餐 |
| 21:00-23:00 | 高 | 睡前、放松 |
| 23:00-1:00 | 中 | 夜猫子 |

**短剧最佳发布时间：**
```
12:00-13:00（午休追剧）
18:00-20:00（下班追剧）
21:00-23:00（睡前追剧）
```

## 二、快手算法详解

### 2.1 核心机制

```
老铁文化 + 社交推荐

特点：
1. 更注重社交关系
2. 老铁粘性高
3. 下沉市场为主
4. 直播带货强
```

### 2.2 核心指标

| 指标 | 权重 | 特点 |
|------|------|------|
| 完播率 | 30% | 与抖音类似 |
| 点赞率 | 20% | 社交属性强 |
| 关注率 | 25% | 老铁关系 |
| 评论率 | 15% | 社区氛围 |
| 分享率 | 10% | 私域传播 |

### 2.3 快手短剧特点

```
1. 剧情更接地气
2. 人物更真实
3. 对白更口语化
4. 节奏可以稍慢
5. 系列感更强
```

## 三、B站算法详解

### 3.1 核心机制

```
内容质量 + 社区氛围

特点：
1. 内容质量优先
2. 用户粘性高
3. 长视频友好
4. 弹幕文化
```

### 3.2 核心指标

| 指标 | 权重 | 特点 |
|------|------|------|
| 完播率 | 25% | 长视频完播难 |
| 三连率 | 30% | 点赞+投币+收藏 |
| 弹幕率 | 20% | 社区互动 |
| 评论率 | 15% | 深度讨论 |
| 分享率 | 10% | 圈层传播 |

### 3.3 B站短剧特点

```
1. 时长可以稍长（3-10分钟）
2. 剧情可以更复杂
3. 制作可以更精良
4. 需要有深度
5. 弹幕互动很重要
```

## 四、微短剧平台算法

### 4.1 核心机制

```
付费转化 + 用户留存

特点：
1. 付费率最重要
2. 完播率次重要
3. 复购率关键
4. 前3集钩子设计
```

### 4.2 核心指标

| 指标 | 权重 | 优化方向 |
|------|------|----------|
| 付费率 | 40% | 付费点设计 |
| 完播率 | 25% | 内容质量 |
| 复购率 | 20% | 系列粘性 |
| 分享率 | 15% | 传播性 |

### 4.3 付费点设计

```
前3集（免费）：
├── 快速建立冲突
├── 展示核心爽点
├── 建立角色魅力
└── 留下悬念

第3集末（付费点）：
├── 最大悬念
├── 最强冲突
├── 最甜瞬间
└── 引导付费

付费后：
├── 及时满足
├── 持续爽点
├── 维持质量
└── 引导下一集
```

## 五、算法优化策略

### 5.1 完播率优化

```
1. 前3秒钩子设计
   - 冲突/反转/悬念
   - 视觉冲击
   - 金句开头

2. 中间节奏控制
   - 每30秒一个刺激点
   - 避免拖沓
   - 信息密度高

3. 结尾钩子设计
   - 悬念/反转
   - 引导下一集
   - 留住观众
```

### 5.2 点赞率优化

```
1. 情感共鸣
   - 触动人心
   - 引发共情
   - 正能量

2. 内容质量
   - 画面精美
   - 剧情精彩
   - 表演到位

3. 金句/名场面
   - 可截图传播
   - 可引用
   - 有记忆点
```

### 5.3 评论率优化

```
1. 话题性设计
   - 争议性话题
   - 讨论空间
   - 观点对立

2. 互动引导
   - 提问
   - 投票
   - 征集意见

3. 彩蛋设计
   - 隐藏细节
   - 考眼力
   - 鼓励发现
```

### 5.4 转发率优化

```
1. 实用性
   - 可学习
   - 可参考
   - 有价值

2. 情感驱动
   - 感动
   - 想分享给某人
   - 代表心情

3. 社交货币
   - 有趣
   - 有品味
   - 有身份认同
```

## 六、算法避坑指南

### 6.1 降权行为

```
❌ 搬运/抄袭
❌ 低质量/模糊
❌ 敏感内容
❌ 频繁删除
❌ 刷量/买粉
❌ 违规导流
```

### 6.2 限流原因

```
1. 内容问题
   - 画面质量差
   - 原创度低
   - 敏感词/画面

2. 账号问题
   - 新号权重低
   - 违规记录
   - 标签不清晰

3. 发布问题
   - 发布时间不对
   - 标题/封面不吸引
   - 标签不准确
```

### 6.3 恢复权重

```
1. 保持更新频率
2. 提高内容质量
3. 避免违规
4. 互动率提升
5. 等待权重恢复
```
